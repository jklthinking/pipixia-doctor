## 1. 项目概述

### 1.1 定位
皮皮虾医生（OpenClaw Doctor）是一个面向 OpenClaw Agent 小白用户的**自诊断自愈 Skill**。它**像顶尖私人医生**一样，能主动发现问题、给出诊断、开出安全药方、按风险等级执行修复。

### 1.2 核心价值
- **零门槛**：小白用户也能用，触发词友好
- **安全优先**：4 级风险策略（L0-L3），绝不越权
- **可沉淀**：所有诊断和修复自动入病历，下次直接复用
- **可自愈**：PCEC 引擎自动 Perceive→Think→Execute→Check

### 1.3 设计哲学
> "让 AI 学会自己看病"——不仅能看病，还能主动预防、自动处方、持续学习。

## 3. 核心模块详解

### 3.1 HEARTBEAT 主动预警（P0-2）

**触发**：定时 + 事件驱动

**L1-L7 分级**：
- L1-L2: info，仅记录
- L3-L4: warning/error，私聊推送
- L5-L7: critical/fatal/panic，飞书群推送

**熔断机制**：
- 连续 3 轮 L4+ error → 降级到静默模式
- 6h+ 无实质进化 → 触发 L4 警告

**通道设计（用户决策 a）**：
- 现有静默规则保留
- HEARTBEAT 走独立通道
- L1-L2 不推送（不破坏用户 SOUL 静默规则）

### 3.2 PCEC 自愈引擎（P0-4）

**4 步循环（K8s 调和模式）**：
1. **Perceive**：调用 doctor_check 收集 findings
2. **Think**：过滤 + 优先级排序
3. **Execute**：L0 自动执行（白名单 6 个动作）
4. **Check**：验证 + 产出三分类

**L0 严格白名单（用户决策 a）**：
- read_health_score（只读）
- summarize_logs（只读）
- mark_case_resolved（写 case）
- update_metrics（写 metrics）
- create_missing_dir（创建目录）
- log_alert（写告警）

**L1+ 必须确认**，绝不自动执行。

**6h 进化停滞熔断**：超过 6h 无新产出 → 强制 L4 警告。

### 3.3 病历管理 + MD5 + 三重备份（P0-5）

**写入流程**：
1. 脱敏（15 种 secret 模式）
2. 写入 .doctor/cases/YYYY-MM-DD.md
3. 计算 MD5
4. 三重备份：原位 + .bak1/ + .bak2/
5. 更新 .doctor/index.json

**校验流程**（启动时）：
1. 读 .doctor/index.json
2. 遍历 .doctor/cases/*.md
3. 校验 MD5 一致
4. 发现篡改：从备份恢复
5. 发现丢失：L4 警告

**设计参照**：Git 的版本控制与不可篡改设计。

### 3.4 锋式十步法（P1-7）

**引用用户 SOUL/AGENTS 里固化的方法论**：

```
1. 剥离情绪 → 2. 定真问题 → 3. 判类型 → 4. 圈边界
5. 拆细碎 → 6. 找根因 → 7. 选方案 → 8. 控风险
9. 明权责 → 10. 常固化
```

**问题类型（per SOUL）**：
- 突发救火（🔴 最高）：先止损再溯源
- 结构优化（🟡 中）：完整 12 阶段
- 决策选择（🟢 较低）：出对比方案
- 协同沟通（🟢 较低）：先共识后执行

### 3.5 日常保健三部曲（P1-10）

| Act | 间隔 | 检查项 |
|-----|------|--------|
| 每日 (daily) | 24h | 心率/病历/日志/git |
| 每周 (weekly) | 7d | 病历/药方/PCEC/内存/集成测试 |
| 每月 (monthly) | 30d | 药方扩增/经验沉淀/架构演进 |

### 3.6 专科门诊（P1-11）

8 个专科（references/specialties/）：
- 网络通信
- 数据库
- 前端 Web
- Python 生态
- Node.js 生态
- 飞书平台
- Git 工作流
- LLM 模型

### 3.7 Health Score 基线（P0-5 增强）

**10 次均值基线**（per PRD 3.3）：
- 异常信号 ≥15 分 → 降级
- 超出预期 ≥10 分 → 正反馈
- 连续 3 轮 error → 熔断
- 6h+ 无进化 → 熔断
- 连续 3 轮 ok → 自动恢复

## 4. 质量保证

### 4.1 三层质量防线

| 防线 | 检查 | 工具 |
|------|------|------|
| 语法门 | Python 语法自检 | `python3 -m py_compile` |
| API 门 | subprocess 全部 list 形式（无 shell=True） | grep + 人工 |
| 功能门 | 22 项集成测试 100% 通过 | `run_tests.py` |

### 4.2 安全脱敏

15 种 secret 模式：
- GitHub PAT (ghp_/gho_/ghu_/ghs_/ghr_)
- AWS Access Key (AKIA)
- OpenAI Key (sk-)
- Stripe (sk_live_/sk_test_)
- Bearer Token
- Token/Password/AppSecret
- Slack (xoxb/xoxa/xoxp/xoxr/xoxs)
- 私钥 (BEGIN PRIVATE KEY)

**在 doctor_record.py 写入时自动脱敏**。

### 4.3 集成测试

22 项测试 100% 通过：
- 11 项 happy path
- 2 项反向 case（redact-secret + route-fallback）
- 4 项 v5.0 Week 1（heartbeat + pcec）
- 2 项 v5.0 Week 2（case-verify × 2）
- 3 项 v5.0 Week 3（ten-step + health-maint）

### 4.4 sub-process 强制范式

所有 `subprocess.run` 都用 list 形式，**禁止** `shell=True`：
- 防御 shell 注入
- 防御命令拼接漏洞
- 例外：必须 shell 特性的场景加 `[SHELL-EXCEPTION]` 注释

## 7. 风险与限制

### 7.1 已知局限

1. **L0 白名单**：只支持 6 个动作，其他 L0 场景需扩展
2. **专科目录**：8 个预定义专科，特殊领域需手动加
3. **OpenClaw 绑定**：当前实现依赖 OpenClaw，v6.0 才通用化
4. **用户本机运行**：尚未在多台 OpenClaw 机器实测

### 7.2 风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| L0 自愈误操作 | 低 | 中 | 严格白名单 |
| HEARTBEAT 误报 | 中 | 低 | 静默规则保护 |
| 病历备份损坏 | 极低 | 高 | 三重备份 + MD5 |
| LLM 幻觉 | 中 | 中 | 强制 source 标注 |

### 7.3 安全边界

- **不执行 L1+** 自动修复（用户决策 a 严格保守）
- **不绕过平台限制**（不绕过飞书 API 限速）
- **不收集敏感信息**（病历自动脱敏）
- **不破坏静默规则**（HEARTBEAT 走独立通道）
